2.3如何定位垃圾?

1.reference count(python 常用)

当有引用指向这个对象时，对其进行计数，当计数变为0时，说明可以对其进行回收。如下图

但RC有缺陷，不能解决循环引用的问题，当出现循环引用，但他们并没有被其它引用指向

故Java使用的是可达性算法

2.Root Searching

根可达或根搜索，顾名思义就是从根开始找

3.GC Algorithms

  • Mark-Sweep (标记清除)
优点 缺点
算法相对简单 两边扫描，效率偏低
存活对象较多情况下效率较高 容易产生碎片，无法存储较大对象
  • Copying(拷贝)
优点 缺点
适用于存活对象较少的情况 空间浪费
只扫描一次，效率提高，没有碎片 移动复制对象，需要调整对象引用
  • Mark-Compact(标记压缩)
优点 缺点
不会产生碎片，方便对象分配 扫描两次
不会产生内存减半 需要移动对象，效率偏低
3.1 Mark-Sweep

3.2Copying

只使用内存的一半空间，将存活的对象移动到另一半内存，再将原有的内存空间全部清掉

3.3Mark-Compact

标记整理/标记压缩

* 各类型垃圾回收器详解

1.Serial & Serial Old(现在基本不使用)

早期JDK使用，究其原因，效率低。随着内存增大，当内存占满，GC做回收，所有任务停止，耗费时间过长，出现卡死

#####2. Parallel Scavenge & Parallel Old(JDK1.8默认)

解决效率问题，引入多线程，即PS&PO

3.ParNew & CMS

但随着内存持续增大，多个线程也不能解决问题。因为当线程过多时，CPU会把资源浪费在管理线程切换上了

并发：工作线程和垃圾回收线程同时执行

2.4 分代模型

old的空间比较大。默认young和old的空间比为1:2

young转换为old是和使用哪种GC有关，默认情况下，CMS为6，其他为5。
